# _skillet_docsite

The user-facing documentation site for `skillet_agent`. It is a [Nextra 4](https://nextra.site/)
static site (Next.js App Router, `output: 'export'`) that builds to plain HTML in `out/` and is
published to GitHub Pages under `/skilled_crew`.

## Local development

```bash
npm install
npm run dev        # typedoc + next dev   → http://localhost:3000
npm run build      # typedoc + next build → static export in out/
npm run deploy     # build, then publish out/ to GitHub Pages
```

`npm run typecheck` runs `tsc --noEmit`.

## What lives where

| Path | What it is |
|---|---|
| `content/**/*.mdx` | **Hand-written** narrative docs (getting started, concepts, configuration, CLI, models, …). |
| `content/**/_meta.ts` | Sidebar order and labels for each section. |
| `content/api/**` | **Auto-generated** API reference. Git-ignored — a build artifact, never edited or committed. |
| `typedoc.json` | TypeDoc config: which agent source to document and where to emit it. |
| `app/`, `mdx-components.tsx`, `next.config.mjs` | Nextra/Next.js wiring. |

## Keeping the docs in sync with `_skillet_agent`

There are **two tracks**, and only one of them is automatic.

### 1. API reference — automatic, cannot drift

`content/api/` is generated by TypeDoc on **every** `npm run dev` and `npm run build` (both run
`npm run typedoc` first). TypeDoc reads the agent's **source** directly:

- entry point: `../_skillet_agent/src/index.ts`
- tsconfig: `../_skillet_agent/tsconfig.json`

So the API reference always reflects whatever `_skillet_agent` currently exports from `src/index.ts`,
including the JSDoc on those symbols. You never hand-edit these pages. To change what appears:

- **Add/remove an API page** → add/remove the export in `_skillet_agent/src/index.ts`.
- **Change a description** → edit the JSDoc on the exported symbol in the agent source.
- **Fix a "referenced but not exported" TypeDoc warning** → export the referenced type from
  `src/index.ts` (this is what issue #258 did).

Because `content/api/**` is git-ignored, a stale checkout simply regenerates it on the next build.
No `dist/` rebuild is needed — TypeDoc compiles the agent's TypeScript source itself.

### 2. Narrative content — manual

Everything else under `content/` is prose written by hand. Nothing detects when the agent's behavior
changes underneath it, so it must be updated deliberately. When you change `_skillet_agent`, check the
page(s) that document that surface:

| When you change in `_skillet_agent`… | Update these pages |
|---|---|
| Providers / model wiring (`src/libs/utils_ai.ts`: `SUPPORTED_PROVIDERS`, `PROVIDER_CLIENT_CONFIGS`, the preflight) | `content/models.mdx`, `content/getting-started/install.mdx` (env table), `content/index.mdx` |
| CLI commands or shared flags (`src/cli.ts`) | `content/cli/commands.mdx` |
| Built-in slash commands (`src/agent_runner/agent_runner.ts` `_handleSlashCommandInternal`) | `content/cli/slash-commands.mdx`, `content/concepts/architecture.mdx` |
| Runtime path layout (`src/libs/skillet_paths.ts`) | `content/concepts/sessions-and-logs.mdx`, `content/concepts/caching-and-cost.mdx`, `content/getting-started/*` |
| `.skilled_crew.yaml` schema (`src/config/skilled_crew_yaml/*_zod.ts`) | `content/configuration/skilled-crew-yaml.mdx`, `content/configuration/json-schema.mdx` |
| `SKILL.md` / `AGENTS.md` / `.command.md` frontmatter (`src/config/{skill,command,agent}/*`) | `content/configuration/{skill-md,agents-md,command-md}.mdx` |
| Default user identity (`src/libs/user_store.ts` `DEFAULT_USER_EMAIL`) | anywhere a user id/email appears (sessions, CLI flags) |
| Environment variables (any new `SKILLET_*` / `*_BASE_URL`) | `content/getting-started/install.mdx` |
| Eval format / grading (`src/evals/*`) | `content/evaluations.mdx` |

A cheap drift check before a docs pass: grep the agent source for the things the docs hard-code, e.g.

```bash
# providers the docs claim vs. the enum
grep -n SUPPORTED_PROVIDERS ../_skillet_agent/src/libs/utils_ai.ts
# CLI subcommands the docs list vs. reality
grep -nE "\.command\('" ../_skillet_agent/src/cli.ts
```

### Sibling-package pages

`content/jobs/**` and `content/web-client/**` document the **separate** `skilled_workflow` and
`_skillet_webclient` packages, not `_skillet_agent`. The durable job/workflow lane (jobs, dispatcher,
schedules, the `/jobs` `/show` `/comment` `/unblock` chat commands) moved out of the agent into
`skilled_workflow` (#247), which embeds the agent runtime as a library and ships its own
`skilled_workflow` binary. Keep those sections in sync with their own packages, not with the agent CLI.
