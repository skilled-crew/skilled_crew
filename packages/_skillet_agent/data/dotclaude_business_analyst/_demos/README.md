# _demos

Demo scenarios for the Business Analyst Agent. Each demo has matching files across folders, identified by a shared name (e.g. `demo_basic_01_create_and_view`).

## Folders

**bin/**
Build tooling. Contains `build.ts` — the script that compiles and runs demos (generates inputs, executes agents, captures outputs).

**descriptions/**
One `.md` file per demo scenario. Human-readable explanations of each demo's goal, context, and expected behavior.

**inputs/**
The input prompts fed to the agent for each demo run — one `.input.txt` per scenario. This is what gets piped into the agent as the user's request.

**outputs/**
The raw agent output captured from each demo run — one `.output.txt` per scenario.

**reports_md/**
The final business reports in Markdown and PDF format generated by the agent during each demo — one `.report.md` and `.report.pdf` per scenario.

**reports_pdf/**
PDF copies of the reports for distribution. Mirrors the PDFs in `reports_md/`.

## Demo naming convention

- `demo_basic_0X` — simple single-skill demos
- `demo_advanced_0X` — multi-step or multi-model scenarios
- `demo_top_0X` — flagship demos (budget reforecast, product launch, cost optimization, board deck prep)
